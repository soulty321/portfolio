Charts

@mui/x-charts v8

Hubble Intent 는 키워드 트렌드·경로분석·여정파인더 등 데이터 시각화가 핵심입니다. 모든 차트는 @mui/x-charts 기반으로 구성되며, 색상은 theme 팔레트의 lineChart · barChart · communityChart · topic · plan 중 용도에 맞는 배열을 인덱스 순으로 매핑합니다.

팔레트 사용 규칙

theme.palette.*
lineChart · 8색
선 기반 시계열 (Trend, Bump, Mini-sparkline). 시리즈 수 ≤ 8.
barChart · 8색
막대 기반 비교 (Gap, Journey Bar). 파스텔톤 — 텍스트 컨트라스트 유지 용이.
topic · 20색
마이키워드 토픽, 버블 차트. 다수의 카테고리를 구분해야 할 때.
communityChart · 26색
클러스터·커뮤니티 그래프 노드용. 20색 이상 필요한 고밀도 네트워크 전용.
plan · 5색
요금제 스택 바 전용 (Trial · Basic · Standard · Advance · Professional).
filter.intent · 4색
INCT 퍼널(Interest · Need · Consideration · Transaction) 고정 매핑. 인덱스 순회 금지.
filter.trend · 2색
증감 지표. Increase #E05265 · Decrease #4F64E1 고정.

구성비·누적 합계. 요금제 점유율 (plan 팔레트), INCT 퍼널 등 고정 카테고리 매핑에 최적.

원칙

Do's & Don'ts
Do
  • 의미가 고정된 값(INCT, Trend, Plan)은 항상 같은 색으로 매핑
  • 시리즈 수 ≤ 8이면 lineChart, 그 이상은 topic
  • Grid 라인은 점선 dashed + grey-200 (데이터 방해 최소화)
  • Tooltip 은 값 오른쪽 정렬 · 서수 tabular-nums
  • Bump 차트 Y축은 반전 (1등이 위)
Don't
  • Intent(INCT)에 lineChart 팔레트 순환 사용 금지
  • 증감 색(Increase/Decrease)을 브랜드 Primary 와 혼용 금지
  • 3 시리즈 이하에서 topic(20색) 사용 금지 — lineChart 사용
  • Grid 라인 실선 사용 금지 (시각적 노이즈)
  • Bar 차트 모서리 radius > 4px 금지 (값 인지 왜곡)
