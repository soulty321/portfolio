## Overview

> **Assets**: (없음)

최대 1,000개 키워드를 30~40초 만에 전략으로 전환하는 AI 에이전트를 설계했습니다.

출시 후 433명의 사용자가 3,410회 분석을 수행했으며, 인당 평균 7.9회 재사용으로 높은 리텐션을 기록했습니다.

## Problem

> **Assets**: `img_source/journeyFinder_projectProblem.svg`

리스닝마인드 사용자들은 방대한 키워드 데이터를 확인할 수 있었지만, 이 데이터를 실제 액션 아이템으로 전환하는 데 큰 어려움을 겪고 있었습니다.

인지적 부하 수준: 1,000 Keywords → 90K+ Tokens, 90% 인지 부하.

### 정량 데이터
분석 대상 키워드 최대 1,000개, LLM에 직접 진행 시 약 9만 토큰 이상 발생. 사용자가 직접 분석하기엔 인지적 과부하가 심각한 수준.

### 정성 데이터 (VOC)
"서비스에서 데이터를 보고는 있는데, 이 데이터로 우리 제품에 맞는 전략을 어떻게 세워야 할지 모르겠어요."

### 현재 사용자 대응 방식
엑셀 다운로드 → 직접 정리 / AI 파일 첨부 → 분석 시도 / 반복적인 수작업 → 시간 소요 & 부정확한 인사이트 도출.

## Strategy

> **Assets**: `img_source/journeyFinder_projectStrategy.svg`

연구소 API를 거치는 7~8단계 복잡한 파이프라인 대신, 프론트엔드 화면 데이터를 백엔드로 직접 전달하는 단순하고 빠른 아키텍처를 채택했습니다.

### 단절된 User Journey
- **맥락 단절**: 화면마다 다른 UI, 일관된 분석 불가.
- **What to do 부재**: 데이터 확인은 가능, 액션 도출 불가.

### 핵심 병목 지점
GPT 리뷰는 각 화면마다 분산·파편화되어 있었고, 사용자는 AI가 제시하는 결과를 일방적으로 수용할 수밖에 없었습니다.

### 프롬프트 진화 단계
- **V0.1**: 비구조적 자유형 요청 / 내부 변수명 노출(volume_trend: 0.73) / 답변 길이 미제어 / 환각(Hallucination) 빈발.
- **V0.2**: 상단 공통 블록 도입(전체 구조 파악) / 변수명 한국어 치환 / 기본 길이 제약 추가 / 품질 불안정·가독성 낮음 / 부분 개선·잘림 이슈 잔존.
- **V0.3**: 3단 구조 확립(상단 구조 / 중간 / 하단 전략 제안) / 분석·요약 호출 분리 / 잘림 해결·일관성 확보.
- **V0.4**: 완전한 3단 템플릿 / 내부 변수명 완전 차단 / 별도 전송(테이블 생성) / 환각 최소화 / 40초·안정적 품질 달성.

### Goal & Hypothesis
사용자가 보는 화면 데이터를 즉시 백엔드로 전달하여 RAG화 하면, LLM에 질문 입력 시 맥락 기반 전략 인사이트를 스트리밍으로 제공할 수 있다.

## Solution A

> **Assets**: (에셋 없음)

전 서비스 화면에서 일관되게 동작하는 슬라이드 오버 형태의 사이드바 기획.

### 사이드바 구조 기획 (아코디언 UI)
방대한 텍스트를 섹션별로 접고 펼칠 수 있어 스크롤 압박 없이 필요한 정보만 즉시 확인.

### 기본 Agent 동작 원리
실시간 화면의 데이터를 읽어내고 사용자에게 제공하는 방식.

사용자가 '데이터 인사이트' 버튼 클릭 → 현재 화면 키워드 데이터 수집(최대 1,000개 → CSV 변환 → Backend 전송) → SSE 스트리밍(실시간 출력) → Gemini API 90K+ 토큰 분석 + 구조화 프롬프트(V0.4) 적용 → 평균 40초 이내 완료.

### 답변 구조
1. 정보 탐색 및 효능 검증
2. Top 5 검색 목적 분석
3. 브랜드/논브랜드 Top 5 분석
4. 인사이트

## Solution B

> **Assets**: (에셋 없음)

1,000개 이상의 키워드를 맥락 있는 전략으로 전환하는 프롬프트를 V0.1 → V0.4 단계로 고도화했습니다.

### AS-IS
프롬프트 구조 설계 (V0.1 ~ V0.2): '상단 공통(데이터 구조 파악) — 중단 상세(Top 3 중심 분석) — 하단 공통(전략 제안)'의 3단계 템플릿을 설계했습니다. 초기 단계에서는 전체 데이터를 단일 프롬프트로 처리하며 답변 패턴과 잘림 한계를 파악했습니다.

### TO-BE
청킹 전략 도입 & 품질 고도화 (V0.3 ~ V0.4): 잘림 이슈 해결을 위해 '분석 호출'과 '요약 호출'을 분리하는 청킹 전략을 적용했습니다. 어드민 피드백 루프(좋아요/싫어요 수집)를 기반으로 프롬프트를 반복 개선하며, 전략 인사이트의 정확도와 가독성을 높였습니다.

### Sub-Agent 비교 분석
2개의 데이터 파일을 동시에 전송하여 변화를 대조 분석.

- 과거 시점 데이터(또는 키워드 그룹 A) ↔ 현재 시점 데이터(또는 키워드 그룹 B)를 동시 전송 → 패턴 변화 대조 분석.
- **니즈 변화 분석**: 기간별 검색 의도 변화, 부상/소멸 키워드 탐지.
- **트렌드 이동 감지**: 경쟁사 vs 자사 비교, 시즌성 패턴 분석.

## Impact

> **Assets**: (에셋 없음)

### 정량적 성과
- 키워드 **3,410개** SEO 분석
- 정확도 **85.7%**
- 추천 정확도 **82.5%**

### 정성적 성과
- **대화 세션 진행**: 433명의 고유 사용자 중 헤비 유저 6명이 50회 이상, 43명이 20회 이상 사용하며 높은 재사용률을 입증했습니다.
- **핵심 3개 Finder에 사용 집중**: 3개 핵심 기능에 전체 사용량의 85.7%가 집중되었습니다.
- **글로벌 채택**: 한국 2,814세션(82.5%), 일본 548세션(16.1%)으로 크로스보더 채택을 달성했습니다.

### Lesson Learned
AI 제품의 진짜 성공 지표는 '사용자가 반복해서 돌아오는가'입니다. 프롬프트 아키텍처(V0.1→V0.4)와 신뢰를 만드는 인터페이스가 결합될 때, AI는 '신기한 기능'이 아닌 '업무 도구'가 됩니다.
