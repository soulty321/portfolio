# 토스증권 차트 UI 관찰 정리

## 1. 구조 분석 — 컬러/토큰 체계의 결함

### 1-1. 시리즈 토큰 슬롯 충돌
- price.up (양봉 빨강) = series.ma20 (빨강) — 동일 hue
- series.ma60 (주황) = bb.upper (주황) = bb.lower (주황) — 3중 충돌
- series.ma5 (초록) = bb.middle (초록) — 동일 hue

### 1-2. 상태 토큰 vs 시리즈 토큰 미분리
현재가 라벨·등락률·양봉·MA20 — 네 가지가 같은 빨강. 강조의 의미 분산.

### 1-3. 패널 간 동일 hue 의미 재사용
- 파랑: MACD 본선 / RSI 시그널 — 패널마다 다른 위계
- 주황: MA60·BB상한·MACD 시그널 — 3패널 재등장
- 자주: MA120·RSI — 2패널 재등장

### 1-4. 동일 데이터 = 동일 토큰 원칙 위배
TIGER S&P500에서 MA20 = BB 중간선 = 18,951원인데 MA20은 빨강, BB 중간선은 초록.

### 1-5. 시각적 정보 손실
KODEX 200 보합 구간 확대 시 빨강 양봉이 빨강 MA20에 흡수 → 캔들이 적어 보이는 착시.

### 1-6. 토큰 정책 일관성
- **잘된 부분**: 거래량 바가 캔들과 동일하게 상승=빨강/하락=파랑. MA에서 파랑 제외하여 음봉 의미 보호.
- **깨지는 부분**: RSI 시그널이 파랑 → 가격 패널 오버레이 시 위반. MACD/RSI 설정창 토큰 노출 비일관.

### 1-7. 글로벌 토큰
KODEX와 TIGER 동일 슬롯. 미국 자산까지 한국 컨벤션(상승=빨강) 강제.

### 1-8. 비교 기능
오버레이가 아닌 1·2·3·4 분할 (멀티차트). 각 패널 독립 → 시리즈 ≤8 유지 가능. 좋은 결정.

## 2. 호균님 1차 사용자 관찰 (음성 메모)

### 용어 장벽이 결정적 문제
- "보조지표 매개변수(5/20/60/120)가 일반 사용자에게 미해독 텍스트"
- "투자를 모르는 사람에게는 너무나도 어려운 용어"

### 공감 기반 진단
- "어려운 용어를 그대로 내준 것에 공감 — 저희 서비스도 같은 고민이 있었기에"
- 비판이 아닌 동료 디자이너 시점

### 레이아웃 긍정
- "차트 레이아웃을 사용자가 마음대로 할 수 있는 건 좋은 것 같고"

### BB 분할 후 명확
- 분할 후 볼린저밴드 상한/하한이 더 명확하게 보임
