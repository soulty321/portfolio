## 사례 1: designSystem_project — "4층 컬러 토큰 + 6종 차트 카탈로그"

### 선정 이유
- JD 핵심 매칭: "차트 시인성", "구조화된 UI", "일관성 있는 UX"
- 지원동기의 "시리즈/상태 토큰 분리" 제안의 실전 근거
- `chart_ui_analysis.md`에서 발견한 토스증권 차트 6개 결함을 해결한 경험으로 연결
- 정량 성과: 빌드타임 -98%, 코드 -40%

## 사례 2: pathDual_project — "듀얼뷰 동기화 인터랙션 (특허)"

### 선정 이유
- JD 핵심 매칭: "복잡한 인터랙션 설계", "PC Web", "데이터 비교"
- WTS 환경 복합 데이터 비교 → 토스증권 멀티차트 분할과 직접 대화 가능
- 정량 성과: 검색량 +26.43%, 특허 등록
- 인지심리학 기반 문제 정의 → 논리적 문제 해결 역량 증명

### 슬라이드 스토리보드

| # | 슬라이드 제목 | 핵심 내용 | 비주얼 |
|---|-------------|----------|--------|
| 1 | Cover | "맥락 전환 없는 비교 경험" | 듀얼 캔버스 아이콘 |
| 2 | Context | B2B SaaS 검색 분석 도구, PM 1 + FE 1 + PD 1(리드) | 서비스 스크린샷 |
| 3 | Problem — 정성 | "갤럭시를 보다가 아이폰과 비교하려면 화면을 닫아야 했다" — 채널톡 VOC 70% 비교 요청 | VOC 로그 인용 + 퍼센트 |
| 4 | Problem — 정량 | 캡처/새 창 우회 → 맥락 전환 비용 발생 | AS-IS 사용자 행동 흐름도 |
| 5 | Insight | 인간 단기기억 한계 — 화면 넘기면 앞 숫자를 잊는다 | 인지심리학 다이어그램 (Miller's Law 참고) |
| 6 | Strategy | "물리적으로 반으로 나누고 동기화하자" — 맥락 전환 비용 제거 | 목표: 검색 사용량 +20% |
| 7 | Solution — 듀얼 캔버스 | 좌우 화면이 하나처럼 움직이되, 필요시 연결 해제 가능 | 듀얼뷰 인터랙션 GIF/도식 |
| 8 | Solution — 동기화 | 확대/클릭/스크롤 0.1초 반응, 3가지 비교 모드 | 동기화 메커니즘 다이어그램 |
| 9 | UI 정리 (AS-IS/TO-BE) | 뷰 전환·키워드 목록을 캔버스 외곽으로 이동 → 데이터 가시성 확보 | AS-IS/TO-BE 비교 |
| 10 | Impact | 검색량 +26.43%, 특허 등록 | 대형 수치 + 특허증 이미지 |
| 11 | Lesson | "데이터는 비교할 때 진짜 의미를 갖는다" — 비교 맥락을 기본 레이어로 | 한 줄 인용 |
| 12 | Chart Team 연결 | "토스증권 멀티차트 분할이 시리즈 ≤8 보호하는 좋은 결정 — 패널 간 동기화 크로스헤어/시간축 동기화를 더하면" | 토스증권 멀티차트 → 개선 제안 |

### 예상 질문 & 답변 준비

| 질문 | 답변 방향 |
|------|----------|
| "동기화 0.1초는 어떻게 달성했나?" | FE와 WebSocket vs requestAnimationFrame 비교 논의 → rAF 선택, 디바운스 설정 |
| "3가지 비교 모드의 우선순위는?" | 사용 빈도 데이터 기반 — 동기화 모드 80%, 독립 모드 15%, 오버레이 모드 5% |
| "모바일에서는 어떻게 적용하나?" | 모바일은 스와이프 기반 전환 → PC에서만 듀얼뷰, 모바일은 빠른 전환 UX |
| "특허 출원 과정은?" | UI 설계 → 변리사 미팅 → 기술 명세서 작성 → 청구항 6개 정리 → 등록 |
| "토스증권 멀티차트에 어떻게 적용?" | 패널 간 시간축 동기화 + 크로스헤어 연동, 이미 좋은 분할 구조에 동기화 레이어만 추가 |
