# 이력서 오버레이 — 토스증권 [Chart]

## 소개 (Override)
다양한 데이터를 사용자가 읽을 수 있는 시각 정보로 전환하는 프로덕트 디자이너 문호균입니다. B2B 데이터 분석 서비스에서 검색량, 시계열, 분석 프레임워크 등 성격이 다른 데이터를 6종 차트로 표현하는 시각 체계를 설계하여, 서비스 전체 화면에 일관된 데이터 시각화 규칙을 적용했습니다. PC Web에서 두 데이터를 동시에 비교하는 듀얼뷰 인터랙션을 설계하여 검색량 +26.43% 향상과 특허 1건을 획득했습니다.

AI 파이프라인 기반 CDJ 자동 분석에서 2,050건의 소비자 여정 데이터를 차트와 테이블로 시각화하여 554명의 활성 사용자를 확보했습니다. '이 데이터를 화면에 표시하였을 때 사용자에게 유의미한 지표가 될까?'를 첫 질문으로, 복잡한 수치를 사용자가 즉시 이해할 수 있는 구조로 전환하는 것에 집중합니다.

## Strength (Override)
**데이터 성격에 따른 시각 표현 체계와 6종 차트 설계**
검색량, 시계열, 분석 프레임워크 등 성격이 다른 데이터를 시각적으로 구분하기 위해, 의미가 고정된 정보와 데이터 계열을 구분하는 정보를 4층으로 분리하는 체계를 설계했습니다. 데이터 목적별로 6종 차트(Line, Bar, Bump, Bubble, Sparkline, Network)를 선정하고, 차트 구성 요소의 표준과 원칙을 정의하여 새로운 차트 추가 시에도 일관된 시각 규칙이 유지되도록 했습니다.

**PC Web 복합 데이터 비교 인터랙션 설계 (특허)**
두 키워드의 여정 데이터를 나란히 비교하는 듀얼뷰 동기화 시스템을 설계했습니다. 양측 캔버스의 클릭·스크롤·줌이 0.1초 내에 동기화되며, 인터랙션의 독창성을 인정받아 특허를 등록했습니다. 기능 출시 후 검색량 +26.43% 향상을 달성했습니다.

**AI 기반 대량 데이터 시각화 시스템 설계**
CDJ 자동 분석 솔루션에서 AI가 분류한 2,050건의 소비자 여정 데이터를 차트와 데이터 테이블로 시각화했습니다. 554명의 활성 사용자, AI 정합성 99%를 달성했습니다.

## Application Motivation (Override)
평소 관심 있던 종목을 토스증권 앱에서 직접 살펴보며 보조지표를 켜봤습니다. 지표가 추가될수록 차트 위 선과 색이 늘어나는데, 각 시각 요소가 어떤 정보를 나타내는지 바로 구분되지 않았습니다. 개별 지표의 색은 잘 설계되어 있기에, 조합 시나리오에서의 시각적 구분이 다음 과제라고 느꼈습니다.

'이 데이터를 화면에 표시하였을 때 사용자에게 유의미한 지표가 될까?' 제 설계의 첫 질문입니다. 검색 데이터 기반 B2B 리서치 툴에서 검색량, 시계열 흐름, 분석 프레임워크 등 성격이 다른 데이터를 차트로 표현해야 했습니다. 화면마다 개별 지정하던 색을, 의미가 고정된 정보와 데이터 계열을 구분하는 정보로 나누어 4층 체계를 설계하고 6종 차트에 적용한 결과, 새로운 차트를 추가할 때도 일관된 시각 규칙으로 데이터를 표현할 수 있게 되었습니다.

토스증권이 투자 경험이 적은 사용자도 쉽게 접근할 수 있도록 차트를 설계하고, 멀티차트 분할로 정보량을 제어하는 점에 주목했습니다. 여기에 데이터의 성격에 따라 시각 정보가 명확히 구분되는 체계까지 더해지면, 가격 등락에 감정이 실리는 주가 차트에서도 사용자가 더 차분하게 판단할 수 있습니다.

입사하게 되면, 사용자 시나리오와 함께 보조지표 3개 이상 조합 시나리오를 팀과 함께 분석하여, 지표가 겹쳐도 각 데이터의 시각적 구분이 유지되는 체계를 제안하고 싶습니다.
