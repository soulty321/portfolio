# 이력서 오버레이 — 토스증권 [Chart]

## 소개 (Override)
데이터 시각화의 규칙을 설계하는 프로덕트 디자이너 문호균입니다. B2B 데이터 분석 서비스에서 4층 컬러 토큰 체계(Categorical·Semantic·Plan·Topic)와 6종 차트 카탈로그를 설계하여, 모든 화면의 차트가 일관된 시각 언어로 데이터를 전달하는 시스템을 구축했습니다. PC Web에서 두 데이터를 동시에 비교하는 듀얼뷰 인터랙션을 설계하여 검색량 +26.43% 향상과 특허 1건을 획득했습니다.

AI 파이프라인 기반 CDJ 자동 분석에서 2,050건의 소비자 여정 데이터를 차트와 테이블로 시각화하여 554명의 활성 사용자를 확보했습니다. 복잡한 수치 데이터를 사용자가 즉시 이해하고 비교할 수 있는 구조화된 UI로 전환하는 것에 집중합니다.

## Strength (Override)
**4층 차트 컬러 토큰 체계와 6종 차트 카탈로그 설계**
Categorical 8색·Semantic 의미 고정·Plan 5색·Topic 26색으로 분리된 컬러 체계를 설계했습니다. 파레토 법칙을 적용하여 상위 8색의 선정에 집중했고, 데이터 목적별로 6종 차트(Line, Bar, Bump, Bubble, Sparkline, Network)를 선종하여 Chart Anatomy 표준과 Do's/Don'ts 원칙을 정의했습니다.

**PC Web 복합 데이터 비교 인터랙션 설계 (특허)**
두 키워드의 여정 데이터를 나란히 비교하는 듀얼뷰 동기화 시스템을 설계했습니다. 양측 캔버스의 클릭·스크롤·줌이 0.1초 내에 동기화되며, 인터랙션의 독창성을 인정받아 특허를 등록했습니다. 기능 출시 후 검색량 +26.43% 향상을 달성했습니다.

**AI 기반 대량 데이터 시각화 시스템 설계**
CDJ 자동 분석 솔루션에서 AI가 분류한 2,050건의 소비자 여정 데이터를 차트와 데이터 테이블로 시각화했습니다. 554명의 활성 사용자, AI 정합성 99%를 달성했습니다.

## Application Motivation (Override)
토스증권 앱에서 차트 보조지표를 여러 개 켜봤을 때, 색이 사용자를 혼란시키는 두 장면을 발견했습니다. 가격 상승을 뜻하는 빨강과 지표선의 빨강이 같아서 겹치는 구간에서 둘이 구분되지 않고, 같은 가격을 가리키는 두 선이 서로 다른 색으로 그려집니다. 색이 '데이터가 오르는가 내리는가'가 아니라 '어떤 지표인가'에 묶여 있어서 생기는 구조적 문제입니다.

'이 데이터가 사용자의 다음 행동을 바꾸는가?'가 제 설계의 첫 질문입니다. B2B 데이터 서비스에서 '상승/하락을 뜻하는 색'과 '각 데이터를 구분하는 색'을 구조적으로 분리하는 컬러 체계를 설계했고, 이 원칙을 적용한 기능에서 검색량 +26.43% 향상을 확인했습니다. 색의 역할이 명확해지자, 전문 용어를 모르는 사용자도 색만으로 데이터의 패턴을 읽기 시작했습니다.

토스증권이 차트를 분할해 한 화면의 정보량을 제어하는 설계를 보며, 이미 복잡도를 구조로 풀고 있다고 느꼈습니다. 여기에 색의 역할 분리까지 더하면, 가격 등락이라는 강한 감정이 개입하는 주가 차트에서 사용자가 더 냉정하게 데이터를 읽을 수 있습니다.

입사 첫 달, 보조지표 3개 이상을 동시에 켠 시나리오를 팀과 함께 분석하여, 지표가 겹쳐도 의미가 충돌하지 않는 색 체계를 제안하고 싶습니다.
